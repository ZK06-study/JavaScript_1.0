왜 Boolean이 필요할까?

상황: 로그인 시스템

if (사용자가_로그인했다) {
    대시보드_보여주기();
} else {
    로그인_페이지_보여주기();
}

상황: 폼 검증

if (이메일이_올바르다 && 비밀번호가_유효하다) {
    회원가입_처리();
} else {
                        error_메시지_표시();
}

프로그래밍의 핵심: 조건에 따른 분기 처리

Boolean은 이러한 의사결정을 가능하게 하는 기본 타입입니다.

Boolean 기본 개념

true

참, 켜짐, 1

false

거짓, 꺼짐, 0

코드 예시

// Boolean 리터럴
let isLoggedIn = true;
let isGuest = false;

// typeof 확인
console.log(typeof true);  // 'boolean'
console.log(typeof false); // 'boolean'

논리 연산자

&&

논리곱 (AND)

모두 참일 때만 참

true && true → true
||

논리합 (OR)

하나라도 참이면 참

true || false → true
!

논리부정 (NOT)

참과 거짓을 뒤바꿈

!true → false

실제 활용 예시

// 접근 권한 확인
if (isLoggedIn && hasPermission) {
    showAdminPanel();
}

// 기본값 설정
let userName = inputName || 'Guest';

논리 연산자 진리표

직접 테스트해보기

&& = true

연산자 우선순위

표현식: true || true && false

true || true && false
1. && 연산자가 먼저 계산됩니다
true || false
2. true && false = false
true
3. true || false = true

우선순위 (높음 → 낮음)

1 ! 논리 부정 (NOT)
2 && 논리곱 (AND)
3 || 논리합 (OR)

Truthy & Falsy

JavaScript에서는 Boolean이 아닌 값도 조건문에서 참/거짓으로 평가됩니다.

Falsy 값들

false
0
""
null
undefined
NaN

Truthy 값들

true
1
"hello"
[]
{}
function(){}

Boolean 변환 테스트

단축 평가 (Short-Circuit)

논리 연산자는 결과가 확정되면 나머지 평가를 중단합니다.

AND (&&) 단축 평가

false
&&
console.log("실행 안됨")
= false
첫 번째가 false면 두 번째는 평가하지 않음

OR (||) 단축 평가

true
||
console.log("실행 안됨")
= true
첫 번째가 true면 두 번째는 평가하지 않음

실용적 활용

기본값 설정

let name = userName || 'Guest';

안전한 접근

user && user.profile && user.profile.name;

실무에서의 Boolean 활용

폼 검증 예시

function validateForm(email, password) {
    const isEmailValid = email.includes('@');
    const isPasswordValid = password.length >= 8;
    
    return isEmailValid && isPasswordValid;
}
